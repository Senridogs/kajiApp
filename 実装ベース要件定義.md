# 実装ベース要件定義（Kaji App）

## 1. 文書の目的
- 本書は、`kaji-app` の**現行実装**を基準に、要件を再定義したもの。
- 「理想仕様」ではなく、現時点で動作している仕様・制約・既知ギャップを明文化する。

## 2. 対象と利用前提
- 対象ユーザー: 同一世帯（主に夫婦）で家事を分担する利用者。
- 利用形態: ブラウザ利用 + PWAインストール（スタンドアロン起動）。
- 対応言語: 日本語UI。
- ストア配布は前提にせず、Web配布前提。

## 3. 業務要件（ドメイン要件）
- 家事ごとに「前回実施日時」を基準として次回期限を算出する。
- 期限当日・期限超過を検知し、通知対象とする。
- 実施記録（だれが、いつ、メモ）を保存し、履歴と統計に反映する。
- 世帯単位で通知ポリシーを管理する。

## 4. 機能要件

### 4.1 セッション・利用開始
- セッションはCookie（`kaji_user_id`, `kaji_household_id`）で管理する。
- Cookie属性: `HttpOnly`, `SameSite=Lax`, 有効期限1年、本番環境は `Secure` 付与。
- 未登録時は初回登録画面を表示する。
- 登録APIは `name` 必須（1〜24文字）、`inviteCode` 任意。
- `inviteCode` 未指定時は新規世帯を作成し、6文字大文字英数字の招待コードを発行する（最大5回リトライで一意性保証）。
- 現行UIは招待コード入力を持たず、名前のみで登録する。

### 4.2 家事管理
- 家事の追加・編集・削除（論理削除: `archived = true`）を提供する。
- 項目:
  - タイトル（必須）
  - 間隔日数（1〜365、整数のみ）
  - 大仕事フラグ（デフォルト: false）
  - アイコン（種類・色・背景色）
    - デフォルト: icon=`sparkles`, iconColor=`#202124`, bgColor=`#EAF5FF`
  - 前回実施日（任意。指定時は初期記録をトランザクションで同時作成）
- カスタムアイコンを作成可能。
- カスタムアイコンはブラウザ`localStorage`に保存し、サーバー永続化はしない。
- アイコン選択UI:
  - クイックプリセット6種（色付き）
  - フルピッカー（144アイコン、24件/ページ × 6ページ）
  - カラーパレット（アイコン色11色、背景色11色）

### 4.3 実施記録
- 家事ごとに実施記録を登録できる（メモ任意、最大500文字）。
- 実施日時を任意で指定可能（未指定時は現在時刻）。
- 実施記録登録時に完了通知（Web Push）を世帯へ送信可能（設定が有効な場合）。
- 実施取消は「当日記録のみ」許可する（過去の記録は削除不可）。
- 履歴は直近200件まで取得する。
- 完了通知の送信先で無効なエンドポイント（404/410レスポンス）は自動的に `enabled=false` に更新する。

### 4.4 ホーム画面
- 日付ヘッダーと以下セクションを表示:
  - 今日
  - 明日
  - 大仕事
- 期限計算ロジック:

| 算出値 | 計算方法 |
|--------|---------|
| dueAt | 最終実行日（未実行の場合は作成日）+ intervalDays |
| isDueToday | dueAt が JST 当日開始〜翌日開始の間 |
| isDueTomorrow | dueAt が JST 翌日開始〜翌々日開始の間 |
| isOverdue | dueAt が JST 当日開始より前 |
| overdueDays | 期限超過日数（整数切り捨て） |
| daysSinceLast | 最終実行からの経過日数 |
| doneToday | 当日中に記録が存在 |

- セクション分類:
  - 今日: `isDueToday` または `isOverdue` または `doneToday`
  - 明日: `isDueTomorrow`
  - 大仕事: `isBigTask = true` かつ期限が今日から7日以内
- 0件セクションは見出しごと非表示。
- 全セクション0件時は空状態UI（「近日実施するべきタスクはありません」）を表示し、追加導線を出す。
- 各行で実施登録、履歴表示、（当日実施済みは）取消を行える。

### 4.5 一覧画面
- 未アーカイブ家事を表示（ソート: 大型タスク降順 → 作成日昇順）。
- 各家事にメタ情報（間隔・前回実施・期限情報など）を表示。
- 行選択で編集ボトムシートを開く。
- 各行に編集・削除ボタンを配置。

### 4.6 統計画面
- 期間:
  - 週（7日）
  - 月（30日）
  - 半年（180日）
  - 年（365日）
  - 全期間（制限なし）
  - カスタム（from/to）
- カスタムは `YYYY-MM-DD`、`from <= to` 必須。
- 出力:
  - 家事別実施回数（上位4件を棒グラフ表示、色分け）
  - ユーザー別実施回数（棒グラフ表示）

### 4.7 設定画面
- 世帯通知設定を変更可能:
  - 期限当日通知（ON/OFF、デフォルト: ON）
  - 期限超過後の毎日通知（ON/OFF、デフォルト: ON）
  - 実施完了通知（ON/OFF、デフォルト: ON）
  - 通知時刻リスト（`HH:mm` 形式、デフォルト: `["06:00","20:00"]`）
- 通知時刻のフォーマットは正規表現でバリデーション。
- テスト通知送信を提供する。
- ユーザー名変更（1〜24文字）。
- 世帯招待コードの表示。
- Push 通知の購読ON/OFF切り替え。

### 4.8 Web Push / PWA
- Service Workerを登録し、Push受信時に通知を表示する。
- 通知タップでアプリ（`/`）へ遷移する。
- `manifest.webmanifest` によりPWAインストール可能。
- Push購読フロー:
  1. ブラウザの `Notification.requestPermission()` で許可取得
  2. Service Worker 登録
  3. `pushManager.subscribe()` で VAPID 公開鍵を使い購読
  4. 購読情報を `POST /api/subscriptions` で DB に保存（endpoint で upsert）
- 購読解除時は `enabled: false` に更新（物理削除しない）。

## 5. 通知要件（バッチ）
- `GET /api/cron/reminders` で通知バッチを実行する。
- `CRON_SECRET` が設定されている場合:
  - `Authorization: Bearer <CRON_SECRET>`
  - または `?secret=<CRON_SECRET>`
  のいずれか必須。
- 時刻判定:
  - 現在JSTの `HH:mm`
  - および同時間の `HH:00`
  のいずれかが `reminderTimes` に含まれる世帯を対象とする。
- 対象家事:
  - `notifyDueToday=true` かつ当日期限
  - または `remindDailyIfOverdue=true` かつ期限超過
- 通知内容:
  - タイトル: `家事リマインド`
  - 本文: 上位2件の家事名 + 残件数
- 送信失敗が`404/410`の購読は `enabled=false` に更新する。
- VAPID環境変数が未設定の場合は通知送信をスキップする。
- Vercel Cron（`vercel.json`）では `11:00 UTC` / `21:00 UTC` 実行（JST 20:00 / 06:00）。

## 6. 完了通知要件
- 家事完了記録時に世帯内の有効な購読へ Push 通知を送信する。
- 通知内容:
  - タイトル: `完了: {家事名}`
  - 本文: `{ユーザー名} が記録しました。` + メモがあれば追記
- `notifyCompletion` が `false` の場合は送信しない。

## 7. API要件（現行エンドポイント）

| メソッド | パス | 認証 | 概要 |
|---------|------|------|------|
| GET | `/api/bootstrap` | 不要 | 初期データ（ユーザー/家事/ホーム分類/通知設定）取得。初回時はデモデータ投入。 |
| POST | `/api/register` | 不要 | ユーザー登録、世帯新規作成または招待コード参加。 |
| PATCH | `/api/user` | Cookie | ユーザー名更新。 |
| GET | `/api/chores` | Cookie | 未アーカイブ家事一覧取得（最新記録付き）。 |
| POST | `/api/chores` | Cookie | 家事作成（初期記録オプション、トランザクション）。 |
| PATCH | `/api/chores/:id` | Cookie | 家事部分更新。 |
| DELETE | `/api/chores/:id` | Cookie | 家事アーカイブ（論理削除）。 |
| POST | `/api/chores/:id/record` | Cookie | 実施記録作成 + 完了通知送信。 |
| GET | `/api/records` | Cookie | 履歴取得（最新200件、実行日時降順）。 |
| DELETE | `/api/records/:id` | Cookie | 当日記録のみ削除可能。 |
| GET | `/api/stats` | Cookie | 統計取得（period/from/toクエリ）。 |
| GET | `/api/notification-settings` | Cookie | 通知設定取得。 |
| PATCH | `/api/notification-settings` | Cookie | 通知設定更新。 |
| POST | `/api/subscriptions` | Cookie | Push購読登録（endpoint で upsert）。 |
| DELETE | `/api/subscriptions` | Cookie | Push購読無効化（enabled=false）。 |
| POST | `/api/notifications/test` | Cookie | テスト通知送信。 |
| GET | `/api/cron/reminders` | Bearer/Secret | リマインド通知バッチ実行。 |

## 8. データ要件（Prismaモデル）

### Household
- id (CUID), inviteCode (Unique), reminderTimes (String[]), notifyDueToday, remindDailyIfOverdue, notifyCompletion
- カスケード削除: 配下の User, Chore, ChoreRecord, PushSubscription を連動削除

### User
- id (CUID), householdId (FK), name (1〜24文字)
- カスケード削除: Household 削除時に連動

### Chore
- id (CUID), householdId (FK), title, icon, iconColor, bgColor, intervalDays (1〜365), isBigTask, archived
- カスケード削除: Household 削除時に連動

### ChoreRecord
- id (CUID), householdId (FK), choreId (FK), userId (FK), memo (最大500文字, nullable), performedAt
- インデックス: `(householdId, performedAt)`, `(choreId, performedAt)`, `(userId, performedAt)`
- カスケード削除: Household 削除時に連動

### PushSubscription
- id (CUID), householdId (FK), userId (FK), endpoint (Unique), p256dh, auth, userAgent (nullable), enabled
- カスケード削除: Household 削除時に連動

## 9. デモデータ要件
- 新規世帯の初回 bootstrap 時にデモ家事6件を自動作成する。
- 130日分以上の履歴を擬似生成（ユーザー間で交互配分）。
- 一部記録にはメモを付与。

| 家事 | 間隔 | 種別 | アイコン | 背景色 |
|------|------|------|---------|--------|
| トイレ掃除 | 4日 | 通常 | toilet（カスタム） | #FDECEF（赤系） |
| 食器洗い | 1日 | 通常 | cooking-pot | #FFF7E0（黄系） |
| 洗面台みがき | 2日 | 通常 | droplets | #EAF5FF（青系） |
| 換気扇掃除 | 30日 | 大型 | fan | #E8F5E9（緑系） |
| 洗濯槽クリーニング | 45日 | 大型 | washing-machine | #E8F5E9（緑系） |
| お風呂防カビ | 30日 | 大型 | shower-head | #E8F5E9（緑系） |

## 10. 非機能要件

### パフォーマンス
- bootstrap API で初期データを一括取得（N+1 回避）。
- 統計集計は Prisma `groupBy` を使用。
- 記録履歴は最新200件に制限。

### セキュリティ
- セッション Cookie: `HttpOnly`, `Secure`（本番）, `SameSite=Lax`。
- Cron エンドポイント: Bearer トークンまたはクエリパラメータで認証。
- 入力値バリデーション（文字数、数値範囲、日付形式、時刻形式）。
- APIエラーレスポンスは `{ error: string }` のJSON形式。

### 信頼性
- Push 通知の無効エンドポイント自動検出・無効化（404/410）。
- 招待コード生成のリトライ機構（最大5回）。
- トランザクション使用（家事作成＋初期記録）。

### タイムゾーン
- 全ての日次判定・通知判定はJST（UTC+9）基準で計算。
- JSTオフセット: 9 * 60 * 60 * 1000 ミリ秒。

### 対応環境
- モバイルファーストのレスポンシブデザイン。
- PWA インストール対応。
- Web Push 通知（ブラウザサポート依存、iOS Safari は制限あり）。

### 技術スタック
- Next.js 16 / React 19 / TypeScript / Tailwind CSS v4 / Prisma / PostgreSQL
- UI: Radix UI, Lucide React（アイコン）, Motion（アニメーション）
- Web Push: web-push パッケージ (VAPID)
- デプロイ: Vercel（Cron は Pro プラン推奨）

## 13. 既知ギャップ・制約（実装準拠）
- 招待コード参加機能はAPI実装済みだが、現行UIからは入力できない。
- 認証は簡易Cookieセッションで、外部IdP/多要素認証は未対応。
- 自動テストはユーティリティ中心で、API/UIの網羅的テストは未実装。
- 統計画面の「ユーザー分担率（円グラフ）」は要件として存在するが、現行実装は棒グラフのみ。
- カスタムアイコンはクライアント localStorage 保存のため、デバイス間同期されない。
- 1世帯あたりの想定ユーザー数は2名（UIは2名前提のレイアウト）。
